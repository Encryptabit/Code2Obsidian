  <files>Analysis/Models/MethodInfo.cs, Analysis/Models/TypeInfo.cs, Analysis/Analyzers/AnalysisHelpers.cs</files>
  <action>
**MethodInfo.cs** -- Add four new fields to the sealed record positional parameters:
- `string Namespace` -- the containing namespace (e.g., "Code2Obsidian.Analysis")
- `string ProjectName` -- the MSBuild project name (e.g., "Code2Obsidian")
- `string AccessModifier` -- lowercase access modifier string (e.g., "public", "private")
- `int CyclomaticComplexity` -- computed cyclomatic complexity score (base 1)

Insert these after `DocComment` (at the end of the positional parameter list). Keep the sealed record pattern.

**TypeInfo.cs** -- Add two new fields to the sealed record positional parameters:
- `string ProjectName` -- the MSBuild project name
- `string AccessModifier` -- lowercase access modifier string

Insert these after `DocComment` (at the end). The TypeInfo record already has `Namespace` and `FilePath`, so only these two are new.

**AnalysisHelpers.cs** -- Add a `ComputeCyclomaticComplexity` static method that accepts a `BaseMethodDeclarationSyntax` and returns `int`. Implementation:
1. Start at base value 1
2. Get descendant nodes from `declaration.Body` (if exists) or `declaration.ExpressionBody` (for expression-bodied members via casting to `MethodDeclarationSyntax`). If neither exists (abstract/extern), return 1.
3. Use `DescendantNodes(ShouldDescend)` where `ShouldDescend` prunes traversal into `SimpleLambdaExpressionSyntax`, `ParenthesizedLambdaExpressionSyntax`, `LocalFunctionStatementSyntax`, and `AnonymousMethodExpressionSyntax` -- these have their own complexity and should not inflate the containing method.
4. Count these syntax node types (each adds 1):
   - `IfStatementSyntax`
   - `WhileStatementSyntax`
   - `ForStatementSyntax`
   - `ForEachStatementSyntax`
   - `DoStatementSyntax`
   - `CaseSwitchLabelSyntax`
   - `CasePatternSwitchLabelSyntax`
   - `CatchClauseSyntax`
   - `ConditionalExpressionSyntax`
   - `ConditionalAccessExpressionSyntax`
   - `BinaryExpressionSyntax` when `IsKind(SyntaxKind.LogicalAndExpression)` or `IsKind(SyntaxKind.LogicalOrExpression)` or `IsKind(SyntaxKind.CoalesceExpression)`
   - `SwitchExpressionArmSyntax` when `arm.Pattern is not DiscardPatternSyntax`

Add `using Microsoft.CodeAnalysis.CSharp;` and `using Microsoft.CodeAnalysis.CSharp.Syntax;` to AnalysisHelpers.cs (it currently only imports `Microsoft.CodeAnalysis`).

The `ShouldDescend` method should be a private static method: `private static bool ShouldDescend(SyntaxNode node) => node is not (SimpleLambdaExpressionSyntax or ParenthesizedLambdaExpressionSyntax or LocalFunctionStatementSyntax or AnonymousMethodExpressionSyntax);`

Also handle `ConstructorDeclarationSyntax` and `DestructorDeclarationSyntax` -- these are `BaseMethodDeclarationSyntax` subtypes with `Body` but no `ExpressionBody`. The `Body` path already handles them.

For `AccessorDeclarationSyntax` (property accessors) -- these are NOT `BaseMethodDeclarationSyntax`, so they won't be passed in. No special handling needed.
