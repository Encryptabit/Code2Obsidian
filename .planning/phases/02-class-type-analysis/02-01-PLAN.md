**Add IsUserType to AnalysisHelpers** (modify existing file):
Add a new static method `IsUserType(INamedTypeSymbol? type, IReadOnlySet<string> projectAssemblyNames)` that returns true if the type belongs to a project assembly. Check: type is not null, ContainingAssembly is not null, assembly name is in projectAssemblyNames. Also add a helper `AccessibilityToString(Accessibility)` that maps the enum to lowercase strings ("public", "internal", "protected", "protected internal", "private protected", "private", "" for default).

Also add a static helper `GetDeclarationOrder(ISymbol symbol)` that returns `symbol.DeclaringSyntaxReferences.FirstOrDefault()?.Span.Start ?? int.MaxValue` for sorting members by source position.

**Create TypeAnalyzer** (new file Analysis/Analyzers/TypeAnalyzer.cs):
Implement `IAnalyzer` with `Name => "TypeAnalyzer"`. Follow the same project/document iteration pattern as MethodAnalyzer:
1. Iterate C# projects, then documents, get syntax tree and semantic model
2. Find all `TypeDeclarationSyntax` nodes (covers classes, interfaces, records, structs)
3. Get `INamedTypeSymbol` via `model.GetDeclaredSymbol(typeDecl)`
4. Filter with `AnalysisHelpers.IsUserType(symbol, context.ProjectAssemblyNames)`
5. For each qualifying type, extract a TypeInfo:
   - Id: `TypeId.FromSymbol(symbol)`
   - Name: `symbol.Name` (simple name, no generics in file name). For generic types, use `symbol.Name` only (e.g., "MyClass" not "MyClass<T>") -- this is the wikilink-safe and filename-safe name.
   - FullName: `symbol.ToDisplayString()`
   - Namespace: `symbol.ContainingNamespace?.ToDisplayString() ?? ""`
   - Kind: Map `symbol.TypeKind` to TypeKindInfo (Class, Interface, Struct; if symbol.IsRecord use Record)
   - FilePath: `symbol.DeclaringSyntaxReferences.FirstOrDefault()?.SyntaxTree.FilePath ?? document.FilePath!`
   - BaseClassFullName: If `symbol.BaseType` is not null and `SpecialType != System_Object`, store `baseType.ToDisplayString()` (fully qualified). If external type (not IsUserType), still store the FullName for display but the emitter knows it has no note. If no base class, null.
   - BaseClassName: If base class exists, store `baseType.Name` (simple name for display). Null otherwise.
   - InterfaceFullNames: `symbol.Interfaces` (NOT AllInterfaces) mapped to `.ToDisplayString()` (fully qualified). Used for wikilink resolution.
   - InterfaceNames: `symbol.Interfaces` mapped to `.Name` (simple names for display).
   - Properties: `symbol.GetMembers().OfType<IPropertySymbol>()`, filter `!IsImplicitlyDeclared`, sort by GetDeclarationOrder, map to PropertyFieldInfo
   - Fields: `symbol.GetMembers().OfType<IFieldSymbol>()`, filter `!IsImplicitlyDeclared && AssociatedSymbol is null` (exclude backing fields), sort by GetDeclarationOrder, map to PropertyFieldInfo
   - Constructors: `symbol.InstanceConstructors`, filter `!IsImplicitlyDeclared`, sort by GetDeclarationOrder, map to ConstructorInfo using `RichSignatureFormat` for DisplaySignature. For each parameter, create ParameterInfo with TypeNoteFullName set to the parameter type's `ToDisplayString()` (fully qualified) if it is a user type (IsUserType check on `param.Type as INamedTypeSymbol`), null otherwise. The emitter uses Sanitize(TypeNoteFullName) for wikilinks.
   - MethodIds: `symbol.GetMembers().OfType<IMethodSymbol>()`, filter via `AnalysisHelpers.IsUserMethod()`, sort by GetDeclarationOrder, map via `MethodId.FromSymbol()`
   - DocComment: Use `AnalysisHelpers.GetMethodDocstring()` adapted for types -- the XML doc API works the same way on INamedTypeSymbol. Create a small helper or extract the summary directly: `symbol.GetDocumentationCommentXml()`, parse for `<summary>` element, normalize.
6. Call `builder.AddType(typeInfo)` (TryAdd handles partial class dedup)
7. Register implementors: for each interface in `symbol.Interfaces`, if that interface is a user type, call `builder.RegisterImplementor(TypeId.FromSymbol(iface), TypeId.FromSymbol(symbol))`

Report progress similarly to MethodAnalyzer (project-level and file-level).

**Define RichSignatureFormat** in AnalysisHelpers (shared between TypeAnalyzer and MethodAnalyzer):
```
public static readonly SymbolDisplayFormat RichSignatureFormat = new(
    typeQualificationStyle: SymbolDisplayTypeQualificationStyle.MinimallyQualified,
    memberOptions:
        SymbolDisplayMemberOptions.IncludeAccessibility |
        SymbolDisplayMemberOptions.IncludeModifiers |
        SymbolDisplayMemberOptions.IncludeType |
        SymbolDisplayMemberOptions.IncludeParameters |
        SymbolDisplayMemberOptions.IncludeRef,
    parameterOptions:
        SymbolDisplayParameterOptions.IncludeType |
        SymbolDisplayParameterOptions.IncludeName |
        SymbolDisplayParameterOptions.IncludeDefaultValue |
        SymbolDisplayParameterOptions.IncludeParamsRefOut,
    genericsOptions:
        SymbolDisplayGenericsOptions.IncludeTypeParameters |
        SymbolDisplayGenericsOptions.IncludeTypeConstraints,
    miscellaneousOptions:
        SymbolDisplayMiscellaneousOptions.UseSpecialTypes
);
```
