       - typeReferences, typeFiles, emittedNotes, typeMetadata, methodIndex, typeIndex: extracted from analysis and emit results.

     - Placeholder for now (implemented in Task 3). Throws NotImplementedException.
   - Public method: `async Task<PipelineResult> RunDryRunAsync(IncrementalState state, CancellationToken ct)`
     - Placeholder for now (implemented in Task 3). Throws NotImplementedException.

  <name>Task 3: Full incremental flow and dry-run in IncrementalPipeline (Cases C/E)</name>
  <files>

  <action>
This task implements the core incremental orchestration inside IncrementalPipeline (NOT Program.cs). All analysis logic stays in the Pipeline layer per INFR-06. Program.cs only updates its routing to call the new methods.

1. Implement `IncrementalPipeline.RunIncrementalAsync(IncrementalState state, CancellationToken ct)`:
   - This replaces the placeholder from Task 2. All orchestration lives HERE, not in Program.cs.
   - **Change detection:** Try GitChangeDetector first. If it throws (RepositoryNotFoundException), fall back to HashChangeDetector.
   - If no changes detected (ChangeSet.Changes is empty): set result metrics to zeros, return PipelineResult with "No changes detected" message. Caller (Program.cs) checks and logs.
   - **Git rename handling** (before analysis): For each rename in changeSet.RenamedPaths, rename vault notes (File.Move) to preserve Obsidian backlinks.

2. **Pass 1 -- Analyze changed files only (inside IncrementalPipeline):**
   - Create MethodAnalyzer and TypeAnalyzer with fileFilter = changeSet.ChangedFilePaths.
   - Call analyzers via the loaded solution from AnalysisContext. The IncrementalPipeline has access to AnalysisContext (passed via constructor or loaded internally).
   - Extract fresh AnalysisResult from pass 1.

3. **Ripple computation (inside IncrementalPipeline):**
   - Call RippleCalculator.ComputeAffectedFiles(changeSet.ChangedFilePaths, freshAnalysis, state).
   - Call RippleCalculator.DetectStructuralRipple(freshAnalysis, changeSet.ChangedFilePaths, state).
   - Union results into the full affected file set.

4. **Pass 2 -- Analyze affected files if ripple expanded (inside IncrementalPipeline):**
   - If affectedFiles is larger than changedFiles: create new analyzers with expanded fileFilter, run again.
   - If same: pass 1 result is sufficient.

5. **Merge with stored data (inside IncrementalPipeline):**
   - Call AnalysisResultMerger.Merge(freshAnalysis, state, affectedFiles) for complete AnalysisResult.

6. **Selective emission (inside IncrementalPipeline):**
   - Create ObsidianEmitter with dirtyFiles = affectedFiles.
   - Create EnrichedResult from merged AnalysisResult (run enricher).
   - Call emitter.EmitAsync with merged EnrichedResult.
   - CRITICAL: Emitter uses full merged AnalysisResult for collision detection, only writes dirty notes.

7. **Stale note cleanup (inside IncrementalPipeline):**
   - Call StaleNoteDetector.FindStaleNotes(state.GetEmittedNotes(), currentEntityIds, affectedFiles).
   - Delete stale note files from disk. Count deletions.

8. **Save state (inside IncrementalPipeline):**
   - Call SaveState helper (from Task 2) with fresh + stored merge.

9. **Populate PipelineResult metrics and return:**
   - WasIncremental = true, FilesSkipped, FilesAnalyzed, NotesWritten, NotesDeleted.

10. **Progress display (inside IncrementalPipeline):**
    - Enumerate total .cs documents in workspace for denominator.
    - Report: "Analyzing {analyzeCount}/{totalCount} files ({skipCount} unchanged)"

11. Implement `IncrementalPipeline.RunDryRunAsync(IncrementalState state, CancellationToken ct)`:
    - Run change detection + ripple computation + stale detection (same as RunIncrementalAsync steps 1-3, 7).
    - Skip emission and state save entirely.
    - Return PipelineResult with counts of what WOULD be regenerated/deleted.
    - If --dry-run without --incremental, Program.cs logs warning and proceeds with full analysis.

12. Update `Program.cs` -- replace Case C placeholder:
    - Case C: call `incrementalPipeline.RunIncrementalAsync(state, ct)`.
    - Case E: call `incrementalPipeline.RunDryRunAsync(state, ct)`.
    - Log results. Program.cs does NOT contain any analysis, ripple, merge, or emission logic -- only CLI routing and progress display.
