This task wires the non-incremental paths and the state save framework. Task 3 builds on this with the full incremental flow.

1. Update `Pipeline/Pipeline.cs`:
   - In Pipeline.RunAsync, set `result.AnalysisResult = analysisResult` and `result.EmitResult = emitResult` before returning, so Program.cs can extract state data.
   - No other structural changes needed -- analyzers and emitter are pre-configured with filters by their callers.

2. Update `Program.cs` -- state save helper:
   - Create a private helper method: `SaveIncrementalState(IncrementalState state, PipelineResult result, IReadOnlySet<string>? affectedFiles, IncrementalState? priorState)`
   - This method extracts state data from the AnalysisResult and EmitResult and calls state.SaveState():
     - fileHashes: For each file in the affected set, compute SHA256 hash of the current file content. Merge with stored hashes (from priorState) for non-affected files. On full analysis, hash all analyzed files.
     - callEdges: From AnalysisResult.CallGraph, extract all edges. For each edge, look up the file path from Methods dict. Merge fresh edges (files in affectedSet) with stored edges (files not in affectedSet). On full analysis, all edges are fresh.
     - typeReferences: From AnalysisResult.Types, for each type, record (typeId, filePath). Also scan Methods for type references via ContainingTypeId.
     - typeFiles: From AnalysisResult.Types, record (typeId, filePath) for all types.
     - emittedNotes: From EmitResult.EmittedNotes, merged with stored notes for non-reanalyzed files.
     - typeMetadata: From AnalysisResult.Types, extract (typeId, baseClass, interfaces, namespace) for structural comparison.
     - methodIndex: From AnalysisResult.Methods, extract (methodId, containingType, filePath).
     - typeIndex: From AnalysisResult.Types, extract (typeId, name, fullName, filePath, kind).
   - State DB path: `Path.Combine(outputDir, ".code2obsidian-state.db")`.
   - Add `.code2obsidian-state.db*` to the vault's .gitignore (create one if it doesn't exist, or append if not already present).

3. Update `Program.cs` -- Case A (no --incremental flag):
   - Behavior unchanged from prior phases. Full analysis, full emission via existing Pipeline.RunAsync. No state interaction.
   - After pipeline completes, if result is successful, check -- do NOT save state when --incremental is not used. State saving only happens when --incremental or --full-rebuild is active.

4. Update `Program.cs` -- Case B (--incremental with no prior state / first run):
   - Instantiate IncrementalState with DB path.
   - Call state.TryLoad(). If returns false (no DB or corrupt):
     - Log: "No prior state found. Performing full analysis and saving state."
     - Perform full analysis (no file filter) via existing Pipeline.RunAsync.
     - After successful emission, call SaveIncrementalState with affectedFiles = null (full mode).
     - Set result.WasIncremental = true (even though it was full, the flag was on).

5. Update `Program.cs` -- Case D (--full-rebuild):
   - Instantiate IncrementalState with DB path.
   - Call state.DeleteState() to wipe the database and WAL/SHM files.
   - Log: "State wiped. Performing full analysis."
   - Perform full analysis via existing Pipeline.RunAsync.
   - After successful emission, call SaveIncrementalState with affectedFiles = null.
   - Set result.WasIncremental = false (this was explicitly a full rebuild).

6. Update `Program.cs` -- routing logic at top of RunPipelineAsync:
   - If --full-rebuild: execute Case D.
   - Else if --incremental: try load state. If no state -> Case B. If state exists -> Case C (implemented in Task 3, for now placeholder that falls through to full analysis with a TODO comment).

<task type="auto">
  <name>Task 3: Full incremental flow and dry-run (Cases C/E)</name>
  <files>
    Program.cs
  </files>
  <action>
This task implements the core incremental orchestration -- the two-pass analysis, ripple computation, selective emission, stale cleanup, and dry-run mode. It replaces the Case C placeholder from Task 2.

1. Update `Program.cs` -- Case C (--incremental with existing state):
   - Load state: IncrementalState.TryLoad returns true with commitSha.
   - Detect changes: Try GitChangeDetector first. If it throws (not a git repo / RepositoryNotFoundException), fall back to HashChangeDetector.
   - If no changes detected (ChangeSet.Changes is empty): log "No changes detected. Vault is up to date." and return exit code 0 immediately.
   - **Git rename handling** (before analysis):
     - If changeSet.RenamedPaths has entries: for each old->new rename, check if a note exists at the old path in the vault. If so, rename the note file (File.Move). This preserves Obsidian backlinks if the vault is in git.

2. **Pass 1 -- Analyze changed files only:**
   - Create MethodAnalyzer and TypeAnalyzer with fileFilter = changeSet.ChangedFilePaths.
   - Call analyzers directly (MethodAnalyzer.AnalyzeAsync + TypeAnalyzer.AnalyzeAsync) without going through Pipeline.RunAsync. This avoids running enrichment + emission for the intermediate pass.
   - Extract fresh AnalysisResult from pass 1.

3. **Ripple computation:**
   - Call RippleCalculator.ComputeAffectedFiles(changeSet.ChangedFilePaths, freshAnalysis, state).
   - Also call RippleCalculator.DetectStructuralRipple(freshAnalysis, changeSet.ChangedFilePaths, state).
   - Union both results into the full affected file set.

4. **Pass 2 -- Analyze affected files (if ripple expanded the set):**
   - If affectedFiles is larger than changedFiles (ripple found neighbors):
     - Create new analyzers with fileFilter = affectedFiles.
     - Call analyzers again. This produces the complete fresh AnalysisResult for ALL affected files.
   - If affectedFiles == changedFiles: pass 1 result is already complete.

5. **Merge with stored data:**
   - Call AnalysisResultMerger.Merge(freshAnalysis, state, affectedFiles) to get complete AnalysisResult.

6. **Selective emission:**
   - Create ObsidianEmitter with dirtyFiles = affectedFiles.
   - Create EnrichedResult from merged AnalysisResult (run enricher if needed, or construct directly).
   - Call emitter.EmitAsync directly with the merged EnrichedResult.
   - CRITICAL: The emitter uses the full merged AnalysisResult for collision detection but only writes notes for dirty files.

7. **Stale note cleanup:**
   - Call StaleNoteDetector.FindStaleNotes(state.GetEmittedNotes(), currentEntityIds, affectedFiles).
   - currentEntityIds = set of all entity IDs in the fresh analysis for reanalyzed files.
   - Delete each stale note file from disk (File.Delete). Count deletions.
   - Set result.NotesDeleted = stale count.

8. **Save state:**
   - After successful emission, call SaveIncrementalState(state, result, affectedFiles, priorState).
   - This merges fresh state data for reanalyzed files with stored data for unchanged files.

9. **Populate PipelineResult metrics:**
   - result.WasIncremental = true
   - result.FilesSkipped = totalSolutionFiles - affectedFiles.Count
   - result.FilesAnalyzed = affectedFiles.Count
   - result.NotesWritten = emitResult.NotesWritten
   - result.NotesDeleted = stale notes count

10. **Progress display integration:**
    - Before running analysis, calculate total files in solution (enumerate all .cs documents in the workspace).
    - Update the Spectre.Console progress description: "Analyzing {analyzeCount}/{totalCount} files ({skipCount} unchanged)"
    - Extract the progress wrapping into a helper so both the full-pipeline path and the direct-analyzer path use Spectre.Console progress.

11. **Implementation note on two-pass approach:**
    For pass 1 and pass 2, call analyzers directly in Program.cs, bypassing Pipeline.RunAsync:
    - Open MSBuildWorkspace and load solution (reuse existing workspace open logic).
    - Call MethodAnalyzer.AnalyzeAsync(solution, progress, ct) and TypeAnalyzer.AnalyzeAsync(solution, progress, ct) with filtered analyzers.
    - For emission: call the emitter directly or use a stripped-down pipeline call.
    The cleanest approach: non-incremental mode uses existing Pipeline.RunAsync (unchanged). Incremental mode orchestrates directly in Program.cs, calling analyzers + enricher + emitter in sequence.

12. Update `Program.cs` -- Case E (--dry-run with --incremental):
    - Perform all steps of Case C up through ripple computation and stale detection, but skip emission entirely.
    - Print: "Would regenerate {N} notes ({M} from changed files, {K} from ripple)."
    - Print: "Would delete {N} stale notes."
    - Print list of affected files.
    - Return exit code 0 without modifying the vault or state.
    - If --dry-run is used without --incremental, log a warning: "--dry-run only works with --incremental. Running full analysis." and proceed normally.
  </action>
