1. Add NuGet packages:
   - `dotnet add package Microsoft.Data.Sqlite --version 8.0.11`
   - `dotnet add package LibGit2Sharp --version 0.31.0`
   Verify the .csproj after adding. Use 8.0.x for Microsoft.Data.Sqlite (NOT 10.0.x) to match the project's .NET 8 target.

2. Create `Incremental/ChangeSet.cs`:
   - `FileChangeKind` enum: `Added`, `Modified`, `Deleted`, `Renamed`
   - `FileChange` sealed record: `string Path`, `string? OldPath` (non-null for renames), `FileChangeKind Kind`
   - `ChangeSet` sealed record: `IReadOnlyList<FileChange> Changes`, `string? CommitSha` (the HEAD commit SHA at detection time, null for hash-only detection), `bool IsFullRebuild` (true when no prior state exists)
   - Add a convenience property `IReadOnlySet<string> ChangedFilePaths` that returns all non-deleted file paths (union of Path for Added/Modified/Renamed) as a case-insensitive HashSet. This is the set used for pipeline file filtering.
   - Add a convenience property `IReadOnlySet<string> DeletedFilePaths` that returns Deleted file paths.
   - Add a convenience property `IReadOnlyDictionary<string, string> RenamedPaths` that maps OldPath -> NewPath for Renamed entries.

3. Create `Incremental/IChangeDetector.cs`:
   - Interface with single method: `ChangeSet DetectChanges(string repoOrProjectPath, IncrementalState? priorState)`
   - When priorState is null, return ChangeSet with IsFullRebuild = true (first run / INFR-05).

4. Create `Incremental/GitChangeDetector.cs`:
   - Implements `IChangeDetector`.
   - Uses `LibGit2Sharp.Repository` to open the repo.
   - If priorState is null OR priorState.CommitSha is null: return ChangeSet with IsFullRebuild = true.
   - Otherwise: compare stored commit tree vs HEAD tree using `repo.Diff.Compare<TreeChanges>(oldTree, newTree, compareOptions)` with `SimilarityOptions.Renames` enabled.
   - ALSO compare HEAD tree vs working directory + index using `repo.Diff.Compare<TreeChanges>(headTree, DiffTargets.Index | DiffTargets.WorkingDirectory)` to catch uncommitted changes.
   - Merge both change sets (committed + working) deduplicating by path.
   - Filter to only `.cs` files (skip non-C# changes).
   - Map LibGit2Sharp `ChangeKind` to our `FileChangeKind` enum.
   - For renames, populate both Path (new) and OldPath (old).
   - Set CommitSha to repo.Head.Tip.Sha.
   - Wrap the entire operation in try-catch: if the repo path is not a git repository (RepositoryNotFoundException), return null or throw a descriptive exception that the caller can catch to fall back to HashChangeDetector.

5. Create `Incremental/HashChangeDetector.cs`:
   - Implements `IChangeDetector`.
   - Fallback detector for non-git repos or dirty working trees where git detection fails.
   - If priorState is null: return ChangeSet with IsFullRebuild = true.
   - Otherwise: enumerate all .cs files under the given path, compute SHA256 hash for each using `SHA256.HashData(stream)` + `Convert.ToHexString()`.
   - Compare against stored hashes from priorState: new files = Added, hash mismatch = Modified, files in state but not on disk = Deleted.
   - Cannot detect renames (hash fallback limitation) -- renames appear as Delete + Add pair.
   - CommitSha = null (no git context).

Namespace for all files: `Code2Obsidian.Incremental`.

  <name>Task 2: SQLite state storage with schema migration</name>
  <files>

  <action>
1. Create `Incremental/StateSchema.cs`:
   - Static class with `EnsureSchema(SqliteConnection connection)` method.
   - Check `PRAGMA user_version` -- if 0 (fresh DB), run MigrateToV1.
   - `MigrateToV1` creates 6 tables in a single transaction (see research schema):
     - `run_state` (id=1 singleton, commit_sha TEXT, run_timestamp TEXT)
     - `file_hashes` (file_path TEXT PK, content_hash TEXT)
     - `call_edges` (caller_id TEXT, callee_id TEXT, caller_file TEXT, callee_file TEXT, PK on caller_id+callee_id)
     - `type_references` (type_id TEXT, file_path TEXT, PK on type_id+file_path)
     - `type_files` (type_id TEXT, file_path TEXT, PK on type_id+file_path)
     - `emitted_notes` (note_path TEXT PK, source_file TEXT, entity_id TEXT)
   - Create indexes per research: idx_call_edges_callee, idx_call_edges_caller_file, idx_call_edges_callee_file, idx_type_references_file, idx_type_files_file, idx_emitted_notes_source, idx_emitted_notes_entity.
   - Set `PRAGMA user_version = 1` at end of migration.
   - Enable WAL mode: `PRAGMA journal_mode = 'wal'` after connection open but before schema check.

2. Create `Incremental/IncrementalState.cs`:
   - Sealed class, NOT IDisposable -- connections are opened and closed per operation (not held open during pipeline).
   - Constructor takes `string dbPath` (absolute path to `.code2obsidian-state.db`).
   - Public property `string DbPath { get; }`.

   **Read methods** (open connection, read, close):
   - `bool TryLoad(out string? commitSha)` -- returns false if DB doesn't exist or is corrupt (catches SqliteException). If DB exists, opens connection, runs EnsureSchema, reads run_state. Returns true with commitSha (or null if no prior run). On ANY exception, returns false (silent full rebuild per user decision).
   - `IReadOnlyDictionary<string, string> GetFileHashes()` -- returns file_path -> content_hash map.
   - `IReadOnlyList<(string CallerId, string CalleeId, string CallerFile, string CalleeFile)> GetCallEdges()` -- returns all stored call edges.
   - `IReadOnlyList<(string TypeId, string FilePath)> GetTypeReferences()` -- returns all type reference entries.
   - `IReadOnlyList<(string TypeId, string FilePath)> GetTypeFiles()` -- returns all type->file mappings (for partial class support).
   - `IReadOnlyDictionary<string, (string SourceFile, string EntityId)> GetEmittedNotes()` -- returns note_path -> (source_file, entity_id).
   - `IReadOnlySet<string> GetCallers(string methodId)` -- returns caller_id set for a given callee_id (one-hop reverse lookup).
   - `IReadOnlySet<string> GetCallees(string methodId)` -- returns callee_id set for a given caller_id.
   - `string? GetFileForMethod(string methodId)` -- looks up caller_file from call_edges where caller_id matches (returns first match).

   **Write methods** (open connection, write in transaction, close):
   - `void SaveState(string? commitSha, IReadOnlyDictionary<string, string> fileHashes, IReadOnlyList<(string CallerId, string CalleeId, string CallerFile, string CalleeFile)> callEdges, IReadOnlyList<(string TypeId, string FilePath)> typeReferences, IReadOnlyList<(string TypeId, string FilePath)> typeFiles, IReadOnlyDictionary<string, (string SourceFile, string EntityId)> emittedNotes)`
     - Opens connection, begins transaction.
     - DELETE all rows from all 6 tables (full state replacement per run -- simplifies logic vs. incremental table updates).
     - INSERT run_state with commitSha and current timestamp.
     - INSERT all file hashes, call edges, type references, type files, emitted notes using parameterized commands.
     - Commit transaction. If any step fails, transaction rolls back automatically.
   - Use `cmd.Parameters.AddWithValue()` for ALL inserts (never string interpolation into SQL).

   **Delete method:**
   - `void DeleteState()` -- deletes the .db file, .db-wal, and .db-shm files if they exist. Used by --full-rebuild.

   **Error handling:**
   - All read methods catch SqliteException and return empty/default values (corrupted state = full rebuild).
   - TryLoad catches ALL exceptions (including file access) and returns false.

Namespace: `Code2Obsidian.Incremental`.
