## Decisions Made
- XDocument.Parse wraps raw LLM text in `<root>` element for safe multi-element XML parsing
- Triple fallback ensures malformed output (common with some providers) degrades gracefully: XML parse -> regex with Singleline -> raw text wrapping
- WebUtility.HtmlDecode applied to all extracted values since LLM output may contain HTML entities
- Tags stored as comma-separated string in SQLite for simplicity; split and trimmed on read
- SanitizeTag strips everything except lowercase alphanumeric and hyphens for YAML safety
- Purpose rendered as bold first line in ## Summary section, followed by summary detail paragraph
