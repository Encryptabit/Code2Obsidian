<objective>
Replace AnsiConsole.Progress() with AnsiConsole.Live() and a two-line rendering approach to eliminate progress bar jitter.

Purpose: INFR-02 requires jitter-free progress display. The current Progress() implementation auto-sizes TaskDescriptionColumn across all rows, so variable-length descriptions cause shared column widths to resize. This plan switches to Live() with fixed-width character bars on separate lines from description text.

Output: ProgressState/StageState types, RenderProgressDisplay helper, Live-based RunWithProgress, all callers updated, CreateProgress deleted.
</objective>

       ```csharp
       var renderLock = new object();
       await AnsiConsole.Live(new Text("Starting..."))
           .AutoClear(false)
           .StartAsync(async ctx => {
               var state = new ProgressState();
               var progress = new Progress<PipelineProgress>(p => {
                   lock (renderLock)
                   {
                       state.Update(p);
                       ctx.UpdateTarget(RenderProgressDisplay(state));
                   }
               });
               result = await operation(progress);
           });
       ```
       The lock serializes concurrent IProgress<T> callbacks from parallel enrichment,
       preventing torn reads and flicker.
