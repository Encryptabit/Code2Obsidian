TryLoad(string configPath)` — returns null if file doesn't exist; reads and deserializes JSON; after deserialization, expands any ApiKey value starting with `$` by looking up `Environment.GetEnvironmentVariable(apiKey.Substring(1))`; if env var is null/empty, leave the `$VAR` string as-is (factory will fail with a clear message).

**ChatClientFactory.cs** (namespace `Code2Obsidian.Enrichment.Config`):
Static class with `static IChatClient CreateFromConfig(LlmConfig config)`:
- Resolve API key: if `config.ApiKey` starts with `$`, throw `InvalidOperationException` explaining the env var was not set.
- Switch on `config.Provider.ToLowerInvariant()`:
  - `"anthropic"`: `new AnthropicClient(new APIKeyAuth(apiKey)).AsIChatClient(config.Model)` — check research for exact constructor. The Anthropic SDK uses `new AnthropicClient(new APIKeyAuth(apiKey))` or `new AnthropicClient(new() { ApiKey = apiKey })`.
  - `"openai"`: `new OpenAIClient(apiKey).GetChatClient(config.Model).AsIChatClient()` — requires `using OpenAI;` and `using Microsoft.Extensions.AI;`.
  - `"ollama"`: `new OllamaApiClient(new Uri(config.Endpoint ?? "http://localhost:11434"), config.Model)` — OllamaApiClient implements IChatClient directly.
  - Default: throw `InvalidOperationException($"Unknown provider '{config.Provider}'. Supported: anthropic, openai, ollama")`.
- IMPORTANT: The enricher knows only `IChatClient`. Provider-specific code lives ONLY here per research anti-pattern guidance.

**StateSchema.cs** — Extend existing schema migration:
- In `EnsureSchema`, after the `if (version < 1)` block, add `if (version < 2) { MigrateToV2(connection); }`.
- Add private `MigrateToV2(SqliteConnection connection)` method:
  ```sql
  CREATE TABLE IF NOT EXISTS summaries (
      entity_id TEXT PRIMARY KEY,
      content_hash TEXT NOT NULL,
      summary_text TEXT NOT NULL,
      model_id TEXT NOT NULL,
      created_at TEXT NOT NULL
  );
  CREATE INDEX IF NOT EXISTS idx_summaries_hash ON summaries(content_hash);
  PRAGMA user_version = 2;
  ```
  Wrap in a transaction like MigrateToV1 does.
- CRITICAL: Existing V1 databases must migrate cleanly. The `IF NOT EXISTS` ensures idempotency. Existing tables are untouched.

**SummaryCache.cs** (namespace `Code2Obsidian.Enrichment`):
Class with constructor taking `string dbPath`. Follow IncrementalState's open-per-operation pattern (NOT IDisposable):
- `string? TryGet(string entityId, string currentContentHash)` — opens connection, calls `StateSchema.EnsureSchema`, queries `SELECT summary_text FROM summaries WHERE entity_id = @id AND content_hash = @hash`. Returns null if no match (cache miss or stale hash). Wraps in try/catch returning null on SqliteException (graceful corruption pattern).
- `void Put(string entityId, string contentHash, string summaryText, string modelId)` — opens connection, calls `StateSchema.EnsureSchema`, executes `INSERT OR REPLACE INTO summaries (entity_id, content_hash, summary_text, model_id, created_at) VALUES (@id, @hash, @text, @model, @time)` with `DateTime.UtcNow.ToString("o")` for created_at. Use parameterized queries.
- `int CountCached(IEnumerable<(string entityId, string contentHash)> entities)` — counts how many entities already have a matching cache entry. Used by cost estimator to determine uncached count.
- Private helper `SqliteConnection OpenConnection()` following IncrementalState pattern: `new SqliteConnection($"Data Source={_dbPath}")`, open, EnsureSchema, return.

**ContentHasher.cs** (namespace `Code2Obsidian.Enrichment`):
Static class with:
- `static string ComputeMethodHash(MethodInfo method, CallGraph callGraph)` — SHA256 hash of: `method.DisplaySignature`, `method.DocComment ?? ""`, sorted callee IDs from `callGraph.GetCallees(method.Id)` prefixed with "calls:", sorted caller IDs from `callGraph.GetCallers(method.Id)` prefixed with "calledby:", and `method.CyclomaticComplexity.ToString()`. Use `System.Security.Cryptography.SHA256.HashData(Encoding.UTF8.GetBytes(input))` and `Convert.ToHexString(hash)`. Build the input as a StringBuilder with AppendLine for each component.
- `static string ComputeTypeHash(TypeInfo type)` — SHA256 hash of: `type.FullName`, `type.DocComment ?? ""`, `type.BaseClassFullName ?? ""`, sorted `type.InterfaceFullNames`, `type.MethodIds.Count.ToString()`, `type.Properties.Count.ToString()`. This captures structural changes that should trigger re-summarization.
- Per research recommendation (Pitfall 3): hash structural content only, NOT raw source text. Formatting changes should NOT invalidate cache.
