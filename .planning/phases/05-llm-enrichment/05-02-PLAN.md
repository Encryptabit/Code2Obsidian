**PromptBuilder.cs** (namespace `Code2Obsidian.Enrichment.Prompts`):
Static class providing prompts optimized for experienced developers per user decision.

- `static string SystemPrompt` — read-only property returning a constant system prompt:
  "You are a code documentation assistant. Write concise, technical summaries for experienced developers. Assume domain knowledge. Reference implementation details. Do not explain basic programming concepts. For simple methods, write a single sentence. For complex methods, write a short paragraph (2-4 sentences). For classes, describe the class's role and primary responsibility, not its member list."

- `static string BuildMethodPrompt(MethodInfo method, AnalysisResult analysis)`:
  Build a user prompt containing:
  - Method signature: `method.DisplaySignature`
  - Doc comment (if present): `method.DocComment`
  - Cyclomatic complexity: `method.CyclomaticComplexity`
  - Callee names (up to 20, sorted): from `analysis.CallGraph.GetCallees(method.Id)`, resolve each to method name via `analysis.Methods` lookup, fall back to MethodId.Value
  - Caller names (up to 20, sorted): from `analysis.CallGraph.GetCallers(method.Id)`, similarly resolved
  - Containing type: `method.ContainingTypeName`
  - Format as a structured prompt like:
    ```
    Summarize this C# method:

    Signature: {signature}
    Class: {containingType}
    Complexity: {complexity}
    {optional: Doc comment: {docComment}}
    Calls: {callee1, callee2, ...}
    Called by: {caller1, caller2, ...}
    ```
  - If no callees/callers, omit those lines rather than showing empty lists.

- `static string BuildTypePrompt(TypeInfo type, AnalysisResult analysis)`:
  Build a user prompt containing:
  - Type name and kind: `type.FullName`, `type.Kind`
  - Doc comment (if present): `type.DocComment`
  - Base class (if present): `type.BaseClassFullName`
  - Interfaces: `type.InterfaceFullNames`
  - DI dependencies: extract from constructors (parameter type names, deduped)
  - Member count: `type.MethodIds.Count`
  - Property/field count: `type.Properties.Count + type.Fields.Count`
  - Format as:
    ```
    Summarize the role and responsibility of this C# {kind}:

    Type: {fullName}
    {optional: Inherits: {baseClass}}
    {optional: Implements: {interfaces}}
    {optional: Dependencies: {diDeps}}
    Members: {memberCount} methods, {propCount} properties/fields
    {optional: Doc comment: {docComment}}
    ```

- `static int EstimatePromptTokens(string systemPrompt, string userPrompt)`:
  Returns `EstimateTokens(systemPrompt) + EstimateTokens(userPrompt)` using the chars/4 heuristic. Helper for CostEstimator.

**CostEstimator.cs** (namespace `Code2Obsidian.Enrichment`):
Static class with:

- `static int EstimateTokens(string text)` — returns `(int)Math.Ceiling(text.Length / 4.0)`. The ~1 token per 4 characters heuristic from research.

- `static (int totalInputTokens, int totalOutputTokens, decimal estimatedCost) EstimateCost(int uncachedCount, int avgInputTokensPerPrompt, int maxOutputTokens, decimal costPerInputToken, decimal costPerOutputToken)`:
  - `totalInputTokens = uncachedCount * avgInputTokensPerPrompt`
  - `totalOutputTokens = uncachedCount * maxOutputTokens`
  - `estimatedCost = totalInputTokens * costPerInputToken + totalOutputTokens * costPerOutputToken`
  - Return all three values as a tuple.

- `static bool ShouldConfirm(int uncachedCount)` — returns true when enrichment is "large" enough to warrant confirmation. Use a threshold of 50 uncached entities as the default. Below 50, auto-proceed; at or above 50, prompt. This is Claude's discretion per user decision.

**LlmEnricher.cs** (namespace `Code2Obsidian.Enrichment`):
Sealed class implementing `IEnricher`.

Constructor parameters:
- `IChatClient client` — the MEAI chat client (from ChatClientFactory)
- `SummaryCache cache` — for cache lookup and storage
- `LlmConfig config` — for MaxConcurrency, MaxOutputTokens, Model
- `IProgress<PipelineProgress>? progress` — optional, for reporting enrichment progress

Public properties:
- `string Name => "LLM Enrichment"` (for pipeline progress display)
- `int InputTokensUsed` — public getter, tracks cumulative input tokens via `Interlocked.Add`
- `int OutputTokensUsed` — public getter, tracks cumulative output tokens via `Interlocked.Add`
- `int EntitiesEnriched` — public getter, count of entities that received LLM summaries
- `int EntitiesCached` — public getter, count of entities served from cache
- `int EntitiesFailed` — public getter, count of entities where LLM call failed

Implementation of `EnrichAsync(AnalysisResult analysis, EnrichedResult enriched, CancellationToken ct)`:

1. Create a `SemaphoreSlim` with `config.MaxConcurrency` (default 3).

2. Build a list of all entities to enrich:
   - For each method in `analysis.Methods`: compute content hash via `ContentHasher.ComputeMethodHash`, check `cache.TryGet(methodId.Value, hash)`. If cache hit, add to `enriched.MethodSummaries` directly and increment `EntitiesCached`. If cache miss, add to uncached methods list.
   - For each type in `analysis.Types`: compute content hash via `ContentHasher.ComputeTypeHash`, check cache. If hit, add to `enriched.TypeSummaries` and increment cached count. If miss, add to uncached types list.

3. Report progress: total entities = methods + types, cached count, uncached count.

4. Process uncached methods in parallel using `Task.WhenAll` with SemaphoreSlim gating:
   - For each uncached method:
     - `await semaphore.WaitAsync(ct)`
     - In try/finally (release semaphore in finally):
       - Build prompt via `PromptBuilder.BuildMethodPrompt`
       - Call `_client.GetResponseAsync` with system prompt + user prompt, `ChatOptions { MaxOutputTokens = config.MaxOutputTokens, Temperature = 0.3f }`
       - Extract `response.Text`; if null/empty, skip
       - Track token usage via `response.Usage?.InputTokenCount` and `OutputTokenCount` using `Interlocked.Add`
       - Store in cache: `cache.Put(methodId, hash, summary, config.Model)`
       - Add to `enriched.MethodSummaries`
       - Increment `EntitiesEnriched`
       - Report progress
     - In catch (Exception): log warning to console, increment `EntitiesFailed`, continue (do NOT rethrow — per Pitfall 5 from research)

5. Process uncached types similarly (same SemaphoreSlim, same error handling):
   - Build prompt via `PromptBuilder.BuildTypePrompt`
   - Same GetResponseAsync/cache/track pattern
   - Store in `enriched.TypeSummaries`

6. After all tasks complete, report final progress with token totals.

IMPORTANT design constraints:
- The enricher knows ONLY about `IChatClient` — no provider-specific code (per anti-pattern guidance).
- Cache lookup MUST precede every LLM call (per Pitfall 3).
- Per-entity try/catch so one failure doesn't abort the run (per Pitfall 5).
- Use `ChatRole.System` and `ChatRole.User` from `Microsoft.Extensions.AI` for message construction.
- Use `new ChatMessage(ChatRole.System, text)` constructor.
  </action>
